## 写在前面

这篇文档用于快速回答一个问题：拿到游戏文件后，应该用哪个模拟器（或运行方式）启动。

最重要的原则只有两条：

- 能用游戏自带原生版本（如 Windows 可执行文件）时，优先原生运行。
- 需要模拟器时，优先选择有公开源码或长期维护的项目。

## 快速对照

Android / iOS 资源会标注 **模拟器引擎**。  
当前标注值为：`KRKR` / `ONS` / `ARTEMIS` / `OTHER`。

这个标注用于说明资源引擎类型，不等于“唯一可用 App 名称”。

- KRKR / ONS / ARTEMIS（Android）：优先使用 Tyranor 最新安装包  
  [Tyranor 2.3.3 APK](https://t.shionlib.com/tools/Tyranor%202.3.3.APK)
- KRKR / ONS（iOS）：在 App Store 自行下载安装对应模拟器。
- OTHER：按资源页备注说明选择运行方式。

新版 Tyranor 已可覆盖 KRKR、ONS、Artemis 这几类常见引擎场景。若资源页明确要求特定补丁或运行方式，请以资源页说明为准。

## 各平台建议

### Windows

- 优先原生运行（游戏自带启动程序）。
- 若资源页标注需特定环境/补丁，请按资源页说明操作。

### Android

- 查看资源卡片上的“模拟器引擎”标注（KRKR / ONS / ARTEMIS / OTHER）。
- KRKR / ONS / ARTEMIS：优先 Tyranor 2.3.3。
- OTHER 或不兼容情况：按资源页备注切换方案。

### iOS

- Tyranor 目前没有 iOS 版本。
- iOS 用户请根据资源标注，在 App Store 自行下载 KRKR / ONS 模拟器。
- 若标注为 ARTEMIS 或 OTHER，请优先参考资源页备注确认可用方案。

## 常见问题

### 明明能启动，但进游戏黑屏/闪退

优先更换同引擎的另一版本的模拟器，再检查资源页备注是否要求特定补丁或者特定版本的模拟器。

### 看不出是什么引擎怎么办？

先查看资源发布页备注；如果没有写明，再根据目录结构和启动文件名判断。无法确认时，在评论区询问发布者。
